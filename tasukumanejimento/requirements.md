# 要件定義書：タスク管理ボード

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | タスク管理ボード |
| 作成日 | 2026年6月27日 |
| 作成者 | マキ |
| バージョン | 1.0 |

### 背景・目的

個人のタスクを「やること・進行中・完了」の3段階で視覚的に管理するため、  
Trello風のカンバンボードをWebアプリとして開発する。

---

## 2. 対象ユーザー

- **利用者**：開発者本人（個人利用）
- **利用環境**：PCブラウザ（Chrome / Edge / Firefox）
- **利用シーン**：日々のタスクや学習の進捗管理

---

## 3. 機能要件

> 機能要件とは「このシステムが何をできなければならないか」を定めたもの。

### 3-1. タスクの追加

| # | 要件 |
|---|------|
| F-01 | タイトル・説明文・期限を入力してタスクを追加できる |
| F-02 | タイトルは必須入力とする。空欄の場合はエラーメッセージを表示する |
| F-03 | 説明文・期限は任意入力とする |
| F-04 | 追加ボタンのクリック、またはEnterキーでタスクを追加できる |
| F-05 | 追加後、入力欄はリセットされる |

### 3-2. タスクの表示

| # | 要件 |
|---|------|
| F-06 | タスクはカード形式で表示する |
| F-07 | カードにはタイトル・説明文・期限を表示する |
| F-08 | 説明文・期限が未入力の場合、その項目はカードに表示しない |
| F-09 | 期限が本日より前の場合、期限を赤字・「⚠️ 期限切れ」と表示する |

### 3-3. タスクの移動

| # | 要件 |
|---|------|
| F-10 | タスクを「やること → 進行中 → 完了」の順に進めることができる |
| F-11 | タスクを前の列に戻すこともできる |
| F-12 | 一番左の列（やること）では「戻す」ボタンを表示しない |
| F-13 | 一番右の列（完了）では「進める」ボタンを表示しない |

### 3-4. タスクの削除

| # | 要件 |
|---|------|
| F-14 | 各カードに削除ボタンを設ける |
| F-15 | 削除前に確認ダイアログを表示する |

### 3-5. データの保存

| # | 要件 |
|---|------|
| F-16 | タスクの追加・移動・削除のたびに自動でブラウザに保存する |
| F-17 | ページを開いたとき、前回保存したタスクを自動で復元する |

---

## 4. 非機能要件

> 非機能要件とは「どのように動くか・品質・制約」を定めたもの。

| # | 項目 | 要件 |
|---|------|------|
| NF-01 | 動作環境 | モダンブラウザ（Chrome / Edge / Firefox の最新版）で動作すること |
| NF-02 | サーバー不要 | HTMLファイルをダブルクリックするだけで起動できること |
| NF-03 | 保存方式 | localStorage を使い、サーバーなしでデータを永続化すること |
| NF-04 | 依存ライブラリ | 外部ライブラリ・フレームワークを使用しないこと |
| NF-05 | レスポンス | タスクの追加・移動・削除は即時反映されること |

## 7. 今後追加を検討している機能（将来要件）

> 現バージョンには含まれないが、将来的に追加を検討している機能。

| # | 機能 | 概要 |
|---|------|------|
| 将来-01 | 優先度 | 高・中・低の優先度をカードに設定できる |

| 将来-03 | ドラッグ＆ドロップ | ボタンではなくマウスのドラッグでカードを移動できる |
